### 1.2 DDD领域驱动设计架构

**架构分层设计：**

```
┌─────────────────────────────────────────────────────────┐
│                  接口层 (Interface Layer)                │
│  - Controller：REST API接口                             │
│  - DTO：数据传输对象                                     │
│  - VO：视图对象                                          │
│  - 参数校验                                              │
│  - 统一响应格式                                          │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                  应用层 (Application Layer)              │
│  - Service：应用服务                                     │
│  - Assembler：对象转换器                                 │
│  - 事务编排                                              │
│  - 业务流程控制                                          │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                  领域层 (Domain Layer)                   │
│  - Entity：领域实体（充血模型）                          │
│  - Value Object：值对象                                  │
│  - Domain Service：领域服务                              │
│  - Repository Interface：仓储接口                        │
│  - Domain Event：领域事件                                │
│  - Specification：规格模式                               │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                  基础设施层 (Infrastructure Layer)        │
│  - Repository Impl：仓储实现                             │
│  - PO：持久化对象                                        │
│  - Mapper：MyBatis-Plus映射器                           │
│  - 外部服务调用                                          │
│  - 消息队列                                              │
│  - 缓存实现                                              │
└─────────────────────────────────────────────────────────┘
```

   ```java
   // 领域实体（充血模型）
   @EqualsAndHashCode(of = "datasetId")
   public class Dataset {
       private Long datasetId;          // 领域标识
       private DatasetCode datasetCode; // 值对象
       private String name;
       private DatasetStatus status;    // 枚举
       private ImageCount imageCount;   // 值对象
       
       // 领域行为
       public void initialize() {
           if (this.status != DatasetStatus.CREATED) {
               throw new DomainException("数据集状态不正确");
           }
           this.status = DatasetStatus.INITIALIZED;
           this.addDomainEvent(new DatasetInitializedEvent(this.datasetId));
       }
       
       public void startCleaning() {
           if (this.status != DatasetStatus.INITIALIZED) {
               throw new DomainException("数据集未初始化");
           }
           this.status = DatasetStatus.CLEANING;
       }
   }
   
   // 值对象
   @Value
   public class DatasetCode {
       String value;
       
       public DatasetCode(String prefix, Long sequence) {
           this.value = prefix + String.format("%06d", sequence);
       }
   }
   
   // 领域服务
   @Service
   public class DatasetDomainService {
       public boolean canMerge(Dataset dataset1, Dataset dataset2) {
           // 领域规则判断
       }
   }
   
   // 仓储接口
   public interface DatasetRepository {
       Dataset findById(Long datasetId);
       void save(Dataset dataset);
       List<Dataset> findByStatus(DatasetStatus status);
   }
   ```

4. **基础设施层实体**
   ```java
   // 持久化对象
   @TableName("dataset")
   public class DatasetPO {
       @TableId(type = IdType.AUTO)
       private Long datasetId;
       private String datasetCode;
       private String name;
       private String description;
       private Short status;
       private Integer totalImageCount;
       private Integer cleanedImageCount;
       private Date createTime;
       private Date updateTime;
   }
   
   // 仓储实现
   @Repository
   public class DatasetRepositoryImpl implements DatasetRepository {
       @Autowired
       private DatasetMapper datasetMapper;
       
       @Override
       public Dataset findById(Long datasetId) {
           DatasetPO po = datasetMapper.selectById(datasetId);
           return DatasetConverter.toEntity(po);
       }
       
       @Override
       public void save(Dataset dataset) {
           DatasetPO po = DatasetConverter.toPO(dataset);
           datasetMapper.insertOrUpdate(po);
       }
   }
   
   // MyBatis-Plus Mapper
   @Mapper
   public interface DatasetMapper extends BaseMapper<DatasetPO> {
       // 自定义查询方法
       @Select("SELECT * FROM dataset WHERE status = #{status}")
       List<DatasetPO> selectByStatus(@Param("status") Short status);
   }
   ```

**DDD核心设计原则：**

1. **充血模型**：领域实体包含业务逻辑和行为，而非贫血模型（只有getter/setter）
2. **聚合根**：Dataset作为聚合根，管理Category、Shard、Image等实体
3. **领域事件**：状态变更时发布领域事件，实现解耦
4. **仓储模式**：通过Repository隔离领域层和数据访问层
5. **规格模式**：复杂的业务规则封装为Specification
6. **分层架构**：严格遵循四层架构，依赖倒置原则

### 1.3 接口自动同步Postman

**Postman自动同步机制：**

```yaml
# postman-sync.yml配置
postman:
  api-key: ${POSTMAN_API_KEY}
  collection-id: ${POSTMAN_COLLECTION_ID}
  workspace-id: ${POSTMAN_WORKSPACE_ID}
  
  # 同步策略
  sync:
    enabled: true
    mode: auto  # auto/manual
    schedule: "0 0 2 * * ?"  # 每天凌晨2点同步
    
  # 接口分组
  groups:
    - name: "数据集管理"
      prefix: "/api/dataset"
    - name: "分类管理"
      prefix: "/api/category"
    - name: "图片管理"
      prefix: "/api/image"
```

**实现方式：**

1. **Swagger集成**
   ```xml

       <groupId>io.springfox</groupId>

       <version>3.0.0</version>
   ```

2. **Postman自动同步服务**
   ```java
   @Service
   public class PostmanSyncService {
       
       @Value("${postman.api-key}")
       private String apiKey;
       
       @Value("${postman.collection-id}")
       private String collectionId;
       
       /**
        * 自动同步接口到Postman
        * 1. 扫描所有Controller接口
        * 2. 生成Swagger文档
        * 3. 转换为Postman Collection格式
        * 4. 调用Postman API更新Collection
        */
       @Scheduled(cron = "${postman.sync.schedule}")
       public void syncToPostman() {
           // 1. 获取Swagger JSON
           String swaggerJson = getSwaggerJson();
           
           // 2. 转换为Postman Collection
           PostmanCollection collection = convertToPostmanCollection(swaggerJson);
           
           // 3. 调用Postman API更新
           updatePostmanCollection(collection);
       }
       
       /**
        * 手动触发同步（支持启动时自动同步）
        */
       @PostConstruct
       public void init() {
           if (syncEnabled) {
               syncToPostman();
               log.info("Postman Collection同步完成");
           }
       }
   }
   ```

## 2. 核心数据模型设计

### 2.1 实体关系

- 数据集(1) → 商品图片分类(N) → 数据分片(N) → 商品图片(N)的层级关系
- 任意数据调整需触发整个数据集统计数据的重新计算

## 3. 核心功能实现要求

### 3.1 数据处理责任链设计

**从数据仓库到四层架构的完整处理流程：**

采用责任链模式串行处理每条数据记录，确保数据处理的完整性和可追溯性。

**责任链处理流程：**

```
数据仓库物化视图
    ↓
1. 数据提取节点
    - 从数据仓库读取原始数据
    - 验证数据格式和完整性
    - 记录提取日志
    ↓
2. 数据转换节点
    - 映射字段到四层架构模型
    - 执行数据标准化处理
    - 记录转换日志
    ↓
3. 数据验证节点
    - 校验数据业务规则
    - 检查数据重复性
    - 记录验证日志
    ↓
4. 数据入库节点
    - 写入临时表
    - 更新统计信息
    - 记录入库日志
    ↓
临时表
```

**责任链节点设计：**

1. **数据提取处理器**
   - 负责从数据仓库获取原始数据
   - 数据源：数据仓库物化视图
   - 处理方式：分批次读取，避免内存溢出
   - 日志记录：记录每批次提取的数据量和时间

2. **数据转换处理器**
   - 负责将原始数据映射到四层架构模型
   - 映射规则：根据业务规则转换字段
   - 处理方式：流式处理，逐条转换
   - 日志记录：记录转换失败的记录和原因

3. **数据验证处理器**
   - 负责验证数据的业务规则和完整性
   - 验证内容：字段格式、业务规则、重复性检查
   - 处理方式：并行验证提高效率
   - 日志记录：记录验证通过的记录数和失败的记录

4. **数据入库处理器**
   - 负责将验证通过的数据写入临时表
   - 入库方式：批量插入提高性能
   - 事务控制：确保数据一致性
   - 日志记录：记录入库成功的记录数和时间

### 3.12 AI智能清洗与评分功能

**AI清洗流程设计：**

```
图片入库
    ↓
触发AI评分任务
    ↓
调用大模型API
    ├─ 图片质量分析
    ├─ 问题识别
    ├─ 清洗建议生成
    └─ 评分计算
    ↓
存储评分结果
    ↓
更新图片状态
    ↓
生成清洗建议
```

**AI评分维度：**

1. **清晰度评分(clarity_score)**
   - 评分范围：0-100分
   - 评分标准：
     - 90-100：非常清晰，细节完整
     - 70-89：清晰，细节基本完整
     - 50-69：一般，部分模糊
     - 30-49：模糊，细节不清晰
     - 0-29：非常模糊，无法辨识
   - 检测内容：图片模糊程度、对焦准确性、噪点情况

2. **完整度评分(completeness_score)**
   - 评分范围：0-100分
   - 评分标准：
     - 90-100：完整，无遮挡无缺失
     - 70-89：基本完整，轻微遮挡
     - 50-69：部分缺失，影响识别
     - 30-49：严重缺失，无法识别
     - 0-29：完全缺失或无效
   - 检测内容：图片完整性、遮挡情况、截断情况

3. **质量评分(quality_score)**
   - 综合评分 = 清晰度评分 × 0.4 + 完整度评分 × 0.6
   - 综合评价：综合考量清晰度和完整度
   - 质量等级：
     - A级(90-100)：优质数据，建议采纳
     - B级(70-89)：良好数据，可以使用
     - C级(50-69)：一般数据，需人工审核
     - D级(30-49)：较差数据，建议废弃
     - E级(0-29)：无效数据，必须废弃

**AI问题识别：**

1. **问题标签类型**

   - 角度不正：检测拍摄角度，计算偏离角度
   - 背景杂乱：分析背景复杂度，评估对主体影响
   - 主体不明确：识别主体位置，评估主体占比

3. **问题严重程度**

### 3.13 数据质量评估功能

**质量评估体系：**

1. **完整性评估(completeness_score)**
   - 计算公式：(非空字段数 / 总字段数) × 100
   - 评估维度：
     - 必填字段完整性
     - 关键字段完整性
     - 图片文件完整性
     - 元数据完整性
   - 问题检测：缺失字段统计、空值比例分析

2. **准确性评估(accuracy_score)**
   - 计算公式：(正确数据数 / 总数据数) × 100
   - 评估维度：
     - 字段格式准确性
     - 数值范围准确性
     - 业务规则准确性
     - 数据类型准确性
   - 问题检测：格式错误、范围异常、类型错误

3. **一致性评估(consistency_score)**
   - 计算公式：(一致数据数 / 总数据数) × 100
   - 评估维度：
     - 字段命名一致性
     - 编码规则一致性
     - 数据格式一致性
     - 业务逻辑一致性
   - 问题检测：冲突数据、不一致字段

4. **及时性评估(timeliness_score)**
   - 计算公式：(及时数据数 / 总数据数) × 100
   - 评估维度：
     - 数据更新及时性
     - 数据采集及时性
     - 数据处理及时性
   - 问题检测：过期数据、延迟数据

5. **唯一性评估(uniqueness_score)**
   - 计算公式：(唯一数据数 / 总数据数) × 100
   - 评估维度：
     - 主键唯一性
     - 业务键唯一性
     - 图片MD5唯一性
   - 问题检测：重复数据、冲突数据

## 5. 业务流程优化设计

### 5.1 数据集生命周期管理

**生命周期阶段：**

```
创建 → 初始化 → 数据导入 → 数据清洗 → 数据审核 → 数据发布 → 数据归档 → 数据删除
```

**各阶段详细说明：**

1. **创建阶段**
   - **准入条件**：用户有创建权限
   - **操作流程**：
     - 用户提交创建申请（名称、描述、业务类型）
     - 系统自动生成数据集编码（前缀+6位序列号）
     - 创建数据集记录，状态设置为"初始化"
     - 创建对应的MinIO存储bucket
     - 发送创建通知
   - **状态转换**：无 → 初始化(0)
   - **异常处理**：编码生成失败、存储创建失败

2. **初始化阶段**
   - **准入条件**：数据集状态为"初始化"
   - **操作流程**：
     - 创建临时表dataset_temp_{dataset_code}
     - 初始化统计信息为0
     - 准备数据导入环境
   - **状态转换**：初始化(0) → 待清洗(1)
   - **异常处理**：临时表创建失败、权限不足

3. **数据导入阶段**
   - **准入条件**：数据集状态为"待清洗"
   - **操作流程**：
     - 从数据仓库物化视图读取数据
     - 责任链处理：提取→转换→验证→入库
     - 写入临时表
     - 从临时表导入到四层架构
     - 更新统计数据（总图片数、分类数、分片数）
     - 触发AI质量评分
   - **状态转换**：待清洗(1) → 清洗中(2)（如自动开始清洗）
   - **异常处理**：数据格式错误、数据量超限、导入中断

4. **数据清洗阶段**
   - **准入条件**：数据集状态为"清洗中"
   - **操作流程**：
     - AI自动清洗：质量<50自动废弃，>80自动采纳
     - 人工清洗：50-80分区间人工审核
     - 批量操作：支持批量废弃/采纳
     - 质量抽检：随机10%复核
   - **状态转换**：清洗中(2) → 已审核(3)
   - **异常处理**：清洗失败、人工审核超时

5. **数据审核阶段**
   - **准入条件**：数据集状态为"已审核"
   - **操作流程**：
     - 提交审核申请
     - 质量抽检（10%随机）
     - 数据完整性检查
     - 管理员审批
     - 审核通过后删除临时表
   - **状态转换**：已审核(3) → 已发布(4)
   - **异常处理**：审核不通过、数据质量问题

6. **数据发布阶段**
   - **准入条件**：数据集状态为"已发布"
   - **操作流程**：
     - 数据安全审核
     - 发布审批
     - 对外开放访问
     - 生成发布报告
   - **状态转换**：已审核(3) → 已发布(4)
   - **异常处理**：发布失败、权限问题

7. **数据归档阶段**
   - **准入条件**：数据集超过保留期限（默认365天）
   - **操作流程**：
     - 检查数据集访问频率
     - 低频数据标记为归档
     - 压缩数据文件
     - 转移到冷存储
     - 保留元数据索引
   - **状态转换**：已发布(4) → 已归档(5)
   - **异常处理**：归档失败、存储空间不足

8. **数据删除阶段**
   - **准入条件**：数据集超过归档期限（默认30天）或用户主动删除
   - **操作流程**：
     - 删除申请（需要审批）
     - 备份数据（保留30天）
     - 删除数据库记录
     - 删除MinIO存储文件
     - 删除缓存
     - 记录删除日志
   - **状态转换**：已归档(5) → 已删除(6)
   - **异常处理**：删除失败、数据恢复

**生命周期管理规则：**

1. **状态转换规则**
   - 只能按顺序正向转换（特殊情况可回退）
   - 每次状态变更记录操作日志
   - 状态变更触发领域事件

2. **回退规则**
   - 清洗中 → 待清洗：清洗过程中断
   - 已审核 → 清洗中：审核不通过
   - 已发布 → 已审核：发布失败

3. **冻结规则**
   - 数据集锁定时禁止状态变更
   - 数据集审核中禁止编辑操作
   - 数据集删除中禁止访问

   ```java
   @Service
   public class ImageCleaningDecisionService {
       
       /**
        * AI自动决策
        */
       public CleaningDecision autoDecision(AiCleanScore score) {
           // 规则1：质量评分>=80，自动采纳
           if (score.getQualityScore() >= 80) {
               return CleaningDecision.autoAdopt(
                   "质量评分优秀，自动采纳",
                   score.getQualityScore()
               );
           }
           
           // 规则2：质量评分<50，自动废弃
           if (score.getQualityScore() < 50) {
               return CleaningDecision.autoDiscard(
                   "质量评分不合格，自动废弃",
                   score.getQualityScore(),
                   score.getProblemTags()
               );
           }
           
           // 规则3：50<=评分<80，需要人工审核
           return CleaningDecision.needManualReview(
               "质量评分一般，需要人工审核",
               score.getQualityScore(),
               score.getProblemTags(),
               score.getCleaningSuggestion()
           );
       }
       
       /**
        * 多条件组合决策
        */
       public CleaningDecision compositeDecision(Image image, AiCleanScore score) {
           // 清晰度<30，直接废弃
           if (score.getClarityScore() < 30) {
               return CleaningDecision.autoDiscard("图片严重模糊", score.getQualityScore());
           }
           
           // 完整度<40，直接废弃
           if (score.getCompletenessScore() < 40) {
               return CleaningDecision.autoDiscard("图片严重缺失", score.getQualityScore());
           }
           
           // 问题标签>=3个，建议人工审核
           if (score.getProblemTags().size() >= 3) {
               return CleaningDecision.needManualReview("存在多个问题，建议人工审核", score.getQualityScore());
           }
           
           return autoDecision(score);
       }
   }
   ```

### 5.3 分类合并与拆分业务规则

**分类合并规则：**

1. **合并条件判定**
   ```java
   // 领域服务：分类合并规则服务
   @Service
   public class CategoryMergeRuleService {
       
       /**
        * 判断是否可以合并
        */
       public MergeEligibility checkMergeEligibility(Long categoryId1, Long categoryId2) {
           Category category1 = categoryRepository.findById(categoryId1);
           Category category2 = categoryRepository.findById(categoryId2);
           
           List<String> reasons = new ArrayList<>();
           boolean canMerge = true;
           
           // 规则1：相似度阈值
           double similarity = calculateSimilarity(category1, category2);
           if (similarity < 0.8) {
               canMerge = false;
               reasons.add(String.format("PLU相似度%.2f低于阈值0.8", similarity));
           }
           
           // 规则2：数据量限制
           int totalCount = category1.getTotalImageCount() + category2.getTotalImageCount();
           if (totalCount > 100000) {
               canMerge = false;
               reasons.add(String.format("合并后数据量%d超过上限100000", totalCount));
           }
           
           // 规则3：状态检查
           if (category1.getStatus() != CategoryStatus.CLEANED || 
               category2.getStatus() != CategoryStatus.CLEANED) {
               canMerge = false;
               reasons.add("分类未完成清洗，不能合并");
           }
           
           // 规则4：锁定检查
           if (category1.getLocked() || category2.getLocked()) {
               canMerge = false;
               reasons.add("分类已锁定，不能合并");
           }
           
           return new MergeEligibility(canMerge, reasons, similarity, totalCount);
       }
       
       /**
        * 计算相似度
        */
       private double calculateSimilarity(Category c1, Category c2) {
           // PLU编码相似度（Levenshtein距离）
           double codeSimilarity = StringSimilarity.levenshtein(c1.getPluCode(), c2.getPluCode());
           
           // PLU名称相似度（余弦相似度）
           double nameSimilarity = StringSimilarity.cosine(c1.getPluName(), c2.getPluName());
           
           // 综合相似度（编码权重0.4，名称权重0.6）
           return codeSimilarity * 0.4 + nameSimilarity * 0.6;
       }
   }
   ```

```java
    public ExportEligibility validateExportConditions(ExportRequest request) {
        List<String> errors = new ArrayList<>();

        if (request.getStatus() != null && request.getStatus() != ImageStatus.CLEANED) {
            errors.add("只能导出已清洗数据");

        if (request.getMinQualityScore() == null) {
            request.setMinQualityScore(70); // 默认最小质量评分
        }
        if (request.getMinQualityScore() < 0 || request.getMinQualityScore() > 100) {
            errors.add("质量评分范围应在0-100之间");
        }
        
        // 规则3：数据量限制
        int estimatedCount = estimateExportCount(request);
        if (estimatedCount > 500000) {
            errors.add(String.format("导出数据量%d超过上限500000，请缩小筛选范围", estimatedCount));
        }
        
        // 规则4：权限检查
        if (!hasExportPermission(request.getUserId(), request.getDatasetId())) {
            errors.add("无导出权限");
        }
        
        // 规则5：频率限制
        if (exceedDailyLimit(request.getUserId())) {
            errors.add("已超过今日导出次数限制");
        }
        
        return new ExportEligibility(errors.isEmpty(), errors, estimatedCount);
    }
```

```java
// 应用服务：导出服务
@Service
public class ExportService {
    
    /**
     * 异步导出
     */
    @Async
    public ExportResult exportAsync(ExportRequest request) {
        // 1. 创建导出任务
        ExportTask task = createExportTask(request);
        
        try {
            // 2. 查询数据（流式查询避免内存溢出）
            Stream<Image> imageStream = imageRepository.streamByConditions(request);
            
            // 3. 创建导出文件
            ExportFileWriter writer = createWriter(request.getFormat());
            writer.open(task.getFilePath());
            
            // 4. 写入数据
            AtomicLong count = new AtomicLong(0);
            imageStream.forEach(image -> {
                writer.write(image, request.getFields());
                count.incrementAndGet();
                
                // 更新进度
                if (count.get() % 1000 == 0) {
                    updateProgress(task.getTaskId(), count.get());
                }
            });
            
            // 5. 完成导出
            writer.close();
            task.complete(count.get());
            
            // 6. 上传到MinIO
            String downloadUrl = uploadToMinIO(task.getFilePath());
            task.setDownloadUrl(downloadUrl);
            
            // 7. 发送通知
            notificationService.sendExportNotification(request.getUserId(), task);
            
            return new ExportResult(true, task.getTaskId(), downloadUrl);
            
        } catch (Exception e) {
            task.fail(e.getMessage());
            throw e;
        }
    }
}
```

```java
// 多级缓存服务
@Service
public class MultiLevelCacheService {
    
    @Autowired
    private Cache localCache;  // L1缓存
    
    @Autowired
    private RedisTemplate redisTemplate;  // L2缓存
    
    /**
     * 多级缓存读取
     */
    public <T> T get(String key, Class<T> type, Supplier<T> loader) {
        // 1. 尝试从L1缓存读取
        T value = localCache.get(key, () -> null);
        if (value != null) {
            return value;
        }
        
        // 2. 尝试从L2缓存读取
        value = (T) redisTemplate.opsForValue().get(key);
        if (value != null) {
            // 回填L1缓存
            localCache.put(key, value);
            return value;
        }
        
        // 3. 从数据源加载
        value = loader.get();
        if (value != null) {
            // 写入L1和L2缓存
            localCache.put(key, value);
            redisTemplate.opsForValue().set(key, value, 1, TimeUnit.HOURS);
        }
        
        return value;
    }
    
    /**
     * 多级缓存写入
     */
    public void put(String key, Object value) {
        // 同时写入L1和L2缓存
        localCache.put(key, value);
        redisTemplate.opsForValue().set(key, value, 1, TimeUnit.HOURS);
    }
    
    /**
     * 多级缓存删除
     */
    public void evict(String key) {
        // 同时删除L1和L2缓存
        localCache.invalidate(key);
        redisTemplate.delete(key);
    }
}
```

```java
@Service
public class CacheBreakdownProtection {
    
    @Autowired
    private RedissonClient redissonClient;
    
    /**
     * 使用分布式锁防止缓存击穿
     */
    public Dataset getDatasetWithLock(String datasetCode) {
        String cacheKey = "dataset:" + datasetCode;
        
        // 1. 查询缓存
        Dataset dataset = cacheService.get(cacheKey, Dataset.class);
        if (dataset != null) {
            return dataset;
        }
        
        // 2. 获取分布式锁
        RLock lock = redissonClient.getLock("lock:" + cacheKey);
        try {
            // 尝试获取锁，最多等待5秒，锁自动过期时间10秒
            if (lock.tryLock(5, 10, TimeUnit.SECONDS)) {
                try {
                    // 3. 双重检查缓存
                    dataset = cacheService.get(cacheKey, Dataset.class);
                    if (dataset != null) {
                        return dataset;
                    }
                    
                    // 4. 查询数据库
                    dataset = datasetRepository.findByCode(datasetCode);
                    if (dataset != null) {
                        cacheService.put(cacheKey, dataset);
                    }
                    
                    return dataset;
                    
                } finally {
                    lock.unlock();
                }
            } else {
                // 获取锁失败，等待一段时间后重试
                Thread.sleep(100);
                return getDatasetWithLock(datasetCode);
            }
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new RuntimeException("获取锁失败", e);
        }
    }
}
```

```java
// Redis分布式锁服务
@Service
@Slf4j
public class RedisDistributedLockService {
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    /**
     * 加锁
     */
    public boolean tryLock(String key, String value, long expire, TimeUnit unit) {
        Boolean result = redisTemplate.opsForValue().setIfAbsent(key, value, expire, unit);
        return Boolean.TRUE.equals(result);
    }
    
    /**
     * 解锁（Lua脚本保证原子性）
     */
    public boolean unlock(String key, String value) {
        String script = 
            "if redis.call('get', KEYS[1]) == ARGV[1] then " +
            "    return redis.call('del', KEYS[1]) " +
            "else " +
            "    return 0 " +
            "end";
        
        RedisScript<Long> redisScript = RedisScript.of(script, Long.class);
        Long result = redisTemplate.execute(redisScript, Collections.singletonList(key), value);
        return result != null && result == 1L;
    }
    
    /**
     * 带重试的锁
     */
    public boolean tryLockWithRetry(String key, String value, long expire, TimeUnit unit, int retryTimes, long retryInterval) {
        for (int i = 0; i < retryTimes; i++) {
            if (tryLock(key, value, expire, unit)) {
                return true;
            }
            
            try {
                Thread.sleep(retryInterval);
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                return false;
            }
        }
        return false;
    }
    
    /**
     * 可重入锁
     */
    public boolean tryReentrantLock(String key, String value, long expire, TimeUnit unit) {
        String currentValue = redisTemplate.opsForValue().get(key);
        
        // 如果是同一个线程持有锁
        if (value.equals(currentValue)) {
            // 重入次数+1
            redisTemplate.opsForValue().increment(key + ":count");
            return true;
        }
        
        // 尝试获取锁
        return tryLock(key, value, expire, unit);
    }
}

@Service
public class CategoryMergeService {
    
    @Autowired
    private DistributedLockService lockService;
    
    /**
     * 合并分类（使用分布式锁）
     */
    public void mergeCategories(Long targetId, Long sourceId) {
        String lockKey1 = "category:lock:" + targetId;
        String lockKey2 = "category:lock:" + sourceId;
        String lockValue = UUID.randomUUID().toString();
        
        try {
            // 获取锁（按顺序获取，避免死锁）
            if (lockService.tryLock(lockKey1, lockValue, 30, TimeUnit.SECONDS)) {
                if (lockService.tryLock(lockKey2, lockValue, 30, TimeUnit.SECONDS)) {
                    try {
                        // 执行合并操作
                        doMerge(targetId, sourceId);
                    } finally {
                        lockService.unlock(lockKey2, lockValue);
                    }
                } else {
                    throw new LockException("获取锁失败: " + lockKey2);
                }
            } else {
                throw new LockException("获取锁失败: " + lockKey1);
            }
        } finally {
            lockService.unlock(lockKey1, lockValue);
        }
    }
```
